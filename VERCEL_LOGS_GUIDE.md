# Vercel 함수 로그 확인 가이드

## Vercel 대시보드에서 로그 확인하는 방법

### 방법 1: 배포 상세 페이지에서 확인 (가장 쉬움)

1. **Vercel 대시보드 접속**
   - [vercel.com](https://vercel.com) 로그인
   - 프로젝트 선택

2. **배포 목록 확인**
   - 프로젝트 페이지에서 "Deployments" 또는 "배포" 섹션 확인
   - 최근 배포 클릭 (가장 위에 있는 배포)

3. **배포 상세 페이지**
   - 배포를 클릭하면 상세 페이지로 이동
   - **"Logs" 또는 "로그"** 탭 클릭
   - 또는 **"Build Logs"** 탭 확인

4. **함수 로그 확인**
   - 로그에서 `/api/auth/register` 관련 오류 찾기
   - 오류 메시지 전체 확인

---

### 방법 2: 실시간 로그 확인

1. **프로젝트 페이지 접속**
2. **"Logs" 또는 "로그"** 탭 클릭 (상단 메뉴)
3. **실시간 로그 확인**
   - 프로덕션에서 회원가입 시도
   - 실시간으로 오류 로그 확인

---

### 방법 3: Functions 탭 (최신 Vercel UI)

1. **프로젝트 페이지 접속**
2. **상단 메뉴 확인:**
   - Overview (개요)
   - Deployments (배포)
   - **Analytics (분석)**
   - **Logs (로그)** ← 여기!
   - Settings (설정)

3. **"Logs" 탭 클릭**
4. **함수 필터링**
   - `/api/auth/register` 검색
   - 또는 시간 필터로 최근 로그 확인

---

## 로그에서 찾아야 할 정보

### 오류 메시지 예시

다음과 같은 오류 메시지를 찾으세요:

```
Register error: Error: Supabase 관리자 클라이언트가 초기화되지 않았습니다.
```

또는

```
Register error: Error: relation "users" does not exist
```

또는

```
Register error: Error: Invalid API key
```

## 스크린샷 가이드

### Vercel 대시보드 구조

```
프로젝트 이름
├── Overview (개요)
├── Deployments (배포) ← 여기서 배포 클릭
│   └── [배포 클릭] → Logs 탭
├── Analytics (분석)
├── Logs (로그) ← 또는 여기서 직접 확인
└── Settings (설정)
    └── Environment Variables (환경 변수)
```

## 대안: 브라우저 개발자 도구 확인

함수 로그를 찾기 어렵다면:

1. **프로덕션 URL에서 회원가입 시도**
2. **브라우저 개발자 도구 열기**
   - `F12` 또는 `Cmd + Option + I`
3. **Network 탭 확인**
   - `/api/auth/register` 요청 찾기
   - 클릭하여 상세 정보 확인
   - Response 탭에서 오류 메시지 확인

---

## 빠른 확인 방법

### 1. 배포 상세 페이지

1. 프로젝트 > Deployments
2. 최근 배포 클릭
3. "Logs" 또는 "Build Logs" 탭 확인

### 2. 실시간 로그

1. 프로젝트 > "Logs" 탭
2. 프로덕션에서 회원가입 시도
3. 실시간 로그 확인

### 3. 브라우저 개발자 도구

1. 프로덕션 URL에서 회원가입 시도
2. `F12` > Network 탭
3. `/api/auth/register` 요청 확인
4. Response에서 오류 메시지 확인

---

## 문제 해결

### 로그를 찾을 수 없다면

1. **브라우저 개발자 도구 사용**
   - Network 탭에서 API 요청 확인
   - Response에서 오류 메시지 확인

2. **Vercel 대시보드 새로고침**
   - 페이지 새로고침 후 다시 확인

3. **다른 배포 확인**
   - 최근 배포가 아닌 다른 배포의 로그 확인
