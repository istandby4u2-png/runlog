## 중요 확인사항

### Supabase 스키마

프로덕션 환경에서도 Supabase 스키마가 필요합니다:

1. **Supabase 대시보드 확인**
   - Table Editor에서 다음 테이블 확인:
     - `users`
     - `courses`
     - `running_records`
     - `likes`
     - `comments`

2. **테이블이 없다면**
   - SQL Editor에서 `SUPABASE_SCHEMA_SETUP.md` 파일의 SQL 스크립트 실행

## 빌드 실패 시

### 일반적인 오류

1. **타입 오류**
   - TypeScript 타입 오류
   - 빌드 로그에서 오류 메시지 확인

2. **린팅 오류**
   - ESLint 오류
   - 대부분 경고이므로 빌드를 막지 않음

3. **환경 변수 오류**
   - 런타임 오류 가능
   - 환경 변수가 모두 추가되었는지 확인

### 해결 방법

1. **빌드 로그 확인**
   - Vercel 대시보드에서 상세 로그 확인
   - 오류 메시지 확인

2. **로컬 빌드 테스트**
   ```bash
   npm run build
   ```
   - 로컬에서 빌드가 성공하는지 확인
