# 404 오류 해결 가이드

## 문제
Network 탭에서 다음 파일들이 404 오류:
- `layout.css`
- `main-app.js`
- `app-pages-internals.js`

## 원인
Next.js 빌드 파일이 생성되지 않았거나 손상되었습니다.

## 해결 방법

### 방법 1: .next 폴더 삭제 후 재시작 (가장 빠름)

터미널에서 다음 명령어를 실행:

```bash
cd "/Users/user/Library/Mobile Documents/com~apple~CloudDocs/NAVER/플레이스/커서"

# 서버 중지 (실행 중이면 Ctrl + C)

# .next 폴더 삭제
rm -rf .next

# 개발 서버 재시작
npm run dev
```

### 방법 2: 완전 정리 후 재시작

```bash
cd "/Users/user/Library/Mobile Documents/com~apple~CloudDocs/NAVER/플레이스/커서"

# 서버 중지
# Ctrl + C

# 캐시 및 빌드 파일 삭제
rm -rf .next
rm -rf node_modules/.cache

# 개발 서버 재시작
npm run dev
```

### 방법 3: 의존성 재설치 (문제가 계속되면)

```bash
cd "/Users/user/Library/Mobile Documents/com~apple~CloudDocs/NAVER/플레이스/커서"

# 서버 중지
# Ctrl + C

# 모든 빌드 파일 및 캐시 삭제
rm -rf .next
rm -rf node_modules/.cache
rm -rf .turbo

# 개발 서버 재시작
npm run dev
```

## 확인 방법

서버가 시작되면:
1. 터미널에 "✓ Ready in ..." 메시지 확인
2. 브라우저에서 http://localhost:3000 접속
3. Network 탭에서 404 오류가 사라졌는지 확인
4. 페이지가 정상적으로 로드되는지 확인

## 예상 결과

정상적으로 시작되면:
- `.next` 폴더가 자동으로 생성됨
- 브라우저에서 페이지가 정상적으로 로드됨
- Network 탭에서 404 오류가 사라짐

## 추가 팁

### 서버가 제대로 시작되지 않는 경우

1. **포트 확인**:
   ```bash
   lsof -i :3000
   ```
   다른 프로세스가 사용 중이면 종료하거나 다른 포트 사용

2. **로그 확인**:
   터미널의 에러 메시지를 확인하세요

3. **Node.js 버전 확인**:
   ```bash
   node --version
   ```
   v18 이상 권장
